# SNHU--CS340
Client/Server Development

  What I have learned throughout this course is that the key to a maintainable, readable, and adaptable program is writing modular code that can be reused and adjusted according to needs, and thoroughly documenting the code as a means to stay organized. Building this project from the ground up in units allowed me to build and test each function of the program individually, and then as a whole as each unit started to come together. It made addressing any bugs/errors much more efficient since most of the time I would catch them before rolling them out. The way the modular CRUD Python module is designed, I could easily reuse this code for any number of projects by simply importing the methods into a new environment.
  The approach to this project is how I like to approach any other. The most important is to understand the problem that you are trying to fix for a client, and then building a solution that addresses very specific needs. An initial outline of how the software should function allows me to break it down into units that are much more approachable to work on. I can address the most basic functions and then apply them to work together on a much higher level. As I mentioned above, a computer scientists job is to solve problems. It ous job to create an efficient and user friendly environment to address said problem, and provide clients like Grazioso Salvare with a product solves their problem and streamlines their operations.
